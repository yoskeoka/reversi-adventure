# Project Plan: Reversi Adventure

## Goal

リバーシ（オセロ）での対戦や局所問題を解きながらストーリーを進めていくゲームを開発する。プレイヤーがゲームをクリアするころにはオセロ協会認定の初段程度の実力が身についている状態を目指す。

## Significance

- リバーシは奥深い戦略ゲームだが、体系的に上達するための教材やゲーム体験が不足している
- ゲーム形式で段階的に学ぶことで、楽しみながら実力を向上できる
- ガチモードとスキルモードの二軸により、幅広いプレイヤー層（初心者〜上級者）が楽しめる

## Requirements

### ゲームモード
- **ガチモード**: ズル無しの正当なリバーシルールでの対戦。純粋な実力勝負
- **スキルモード**: スキル（特殊能力）ありの改変ルールリバーシ。イージーモードとしての位置づけに加え、ガチプレイヤーの気分転換としても機能する

### ゲーム進行
- 対戦や局所問題（定石問題、詰めオセロ等）をクリアすることでストーリーが進行する
- 難易度を段階的に上げ、クリア時にはオセロ初段相当の実力が身につくカリキュラム設計

### リバーシAI
- 最強である必要はない。人間の名人程度の実力があれば十分
- **高速・省メモリ**であること（快適なプレイ体験のため）
- **着手理由の説明機能**: なぜその手が良いのか、理由を説明できることが重要（教育的価値）

### プラットフォーム
- **Steamでのリリースを前提**とする

### 技術スタック
- **ゲームエンジン**: Godot 4.x (GDScript)
- **AIエンジン**: Rust (godot-rust GDExtension経由でGDScriptから呼び出し)
- **Steam連携**: GodotSteam (GDExtension plugin)
- **開発方針**: 全コードをLLM（AI）で生成。コードオンリーワークフローを重視

### 非機能要件
- プレイヤーが快適に遊べるレスポンス速度
- 段階的な学習体験を提供するUI/UX

## Milestones
